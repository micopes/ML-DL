머신러닝

명시적인 프로그래밍 없이 컴퓨터가 학습하는 능력을 갖추게 하는 연구분야.

작업 T(문제), 성능 측정 P(정확도), 경험 E(훈련 데이터)

왜 머신러닝을 사용하는가
전통적인 접근 방법에서는 문제가 단순하지 않아 규칙이 점점 길고 복잡해진다.
알려진 알고리즘이 없는 경우
대용량의 데이터를 학습하여 보이지 않는 패턴을 파악하는 것.

- 지도, 비지도, 준지도, 강화학습

지도 학습 : 훈련 데이터에 label(레이블)이라는 원하는 답이 포함됨.
분류(Classification) : 어떤 클래스에 속해있는지를 분류
회귀(Regression) : 연속된 값들에서 input 을 통해서 어떤 output이 나오는지.

비지도 학습 : 훈련 데이터에 label이 없는 경우. 답이 없는 경우임.
군집(Clustering)
어떻게 중요한 것들을 추려낼 것인가(차원 축소)
차원 축소에서 특성 추출이라는 기능이 있음.( 특성을 조합 )

clustering -> 차원 축소 -> 의미 있는 군집을 강조 t-SNE 시각화

일반화한 범주에서 벗어나는 경우 : 이상.(이상 탐지)

준지도 학습 : 레이블이 없는 많은 데이터 + 레이블이 있는 일부 데이터

강화 학습 : 학습하는 시스템인 agent가 환경(environment)를 관찰해서 행동하고 그 결과로 보상을 받음
물(점수 잃음)<- agent -> 불(점수 얻음)
가장 많은 점수를 얻도록 학습하는 것.

- 온라인 학습, 배치 학습

배치 학습 : 
- 시스템이 점진적으로 학습할 수 없는 오프라인 학습
- 가용한 데이터를 모두 사용해 훈련해야 함.
시간이 오래 걸려 모델을 구성하면 사용만 가능함.

온라인 학습 :
데이터를 순차적으로 주입하여 시스템을 훈련시킨다.
나쁜 데이터를 넣으면 시스템 성능이 점진적으로 감소함.
빠른 변화에 적응해야 하는 경우 유리

기존의 데이터와 새로운 데이터 간에 중요성을 고려하여 배치학습을 할 지 온라인 학습을 할 지를 결정해야 함.

- 사례 기반 학습, 모델 기반 학습

사례 기반 학습
- 시스템에 사례를 기억함으로써 학습함
- 유사도(similarity)를 측정하여 새로운 데이터에 일반화 함.

모델 학습 기반
기준선을 치는 것.
새로운 샘플에 대해서 이 모델이라는 기준 선을 이용하여 판단함.

**
머신러닝의 주요 도전과제
나쁜 데이터
- 충분하지 않은 양의 훈련 데이터
- 대표성 없는 훈련 데이터(특이한 데이터 존재)
- 낮은 품질의 데이터
- 관련 없는 특성

나쁜 알고리즘
- Overfitting 데이터 많이 모으고, 잡음을 제거, parameter 수가 작은 모델을 사용.
- Underfitting 모델이 너무 단순하여 학습이 제대로 되지 않는 것. parameter가 더 많은 모델을 사용해야 함.
